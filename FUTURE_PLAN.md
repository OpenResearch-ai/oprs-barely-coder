# OpenResearch.ai — Future Plan

> 바이브 코딩으로 만드는 AI 커뮤니티. 에이전트가 개발하고 커뮤니티가 방향을 정한다.

---

## 현재 구현된 기능 (2026.03)

### 커뮤니티
- 포스트 CRUD (글 작성/조회/삭제)
- 카테고리 필터 (바이브코딩, AI/LLM, 쇼케이스, 리소스, 질문, 서비스제안)
- 작품별 필터 (oo.ai, o talk, openresearch.ai)
- 업보트 (DB 영속, 새로고침 유지)
- 댓글 + 대댓글 + 이미지 첨부
- 작성자 클릭 → 해당 작성자 글 필터
- 키워드 검색 (클라이언트 사이드)

### AI 기능
- Gemini 3.1 Flash Lite 기반 챗봇 (툴 콜링: 페이지 이동, 검색, 필터, 글쓰기)
- AI 콘텐츠 모더레이션 (글/댓글/이미지 멀티모달)
- AI 자동 크롤링 (HN, Reddit, GeekNews) → 텔레그램 컨펌
- AI 스프린트 집계 (커뮤니티 피드백 → 스프린트 자동 생성)
- URL 자동완성 (OG + 이미지 분석 → 한국어 초안)

### 운영
- Supabase Auth (Google OAuth + 이메일)
- 24시간 유저 밴 시스템
- Rate limiting (채팅/글/댓글)
- 텔레그램 봇 연동 (모더레이션 승인/거절, 크롤 컨펌, 통계)
- Vercel Cron (6시간 싱크, 30분 크롤, 1시간 통계)
- Google Analytics

## 우선순위 높음 🔴

### 1. 검색 고도화
- **현재**: 클라이언트 사이드 키워드 매칭 (로드된 20개만 검색)
- **개선**: Supabase Full-Text Search (`to_tsvector`) 또는 Algolia
- **왜**: 글이 늘어날수록 현재 방식 한계

### 2. 유저 프로필 페이지 고도화
- 내가 쓴 글 목록
- 내가 추천한 글 목록
- 활동 히스토리
- 팔로우 기능

### 3. 알림 시스템
- 내 글에 댓글 달렸을 때 인앱 알림
- 웹 푸시 알림 (Service Worker + VAPID)
- 텔레그램 알림 연동 (선택적)

### 4. 포스트 상세 페이지 공유
- OG 메타태그 동적 생성 (제목/이미지/설명)
- 트위터/링크드인 공유 버튼

## 아키텍처 개선 포인트

### 성능
- `CommunityFeed` 클라이언트 사이드 검색 → 서버 사이드 FTS로 전환
- 이미지 CDN 최적화 (Supabase Storage + Next.js Image)
- React `useSWR` 또는 `TanStack Query`로 캐싱 개선

### 안정성
- API Rate Limit을 Redis 기반으로 전환 (현재 Supabase DB)
- 에러 모니터링 (Sentry)
- 타입 안전성: Supabase CLI 타입 자동 생성 (`supabase gen types`)

### 보안
- CSP 헤더 설정
- CORS 명시적 설정
- 이미지 업로드 악성 파일 탐지 강화

### 개발 경험
- Storybook으로 컴포넌트 문서화
- E2E 테스트 (Playwright)
- GitHub Actions CI/CD
